First column seems useless (it looks like a row numbering)

We can see that column (category) contains 2 interesting information: age category and gender. Therefore, we can create 2 new columns for gender & age In addition, we add a column status (1 = finisher; 0 = DNF / did not finish) based on the presence or not of a time in the column Arrive

data_utmb17 <-data_utmb17 |>
  mutate(gender = case_when(
    endsWith(category, " H") ~ "Male",
    endsWith(category, " F") ~ "Female"),
    age = substring(data_utmb17$category, first=1, last=2),
    status = case_when(time != 'NA' ~ 1, TRUE ~ 0),
    .after ="category")

We can observe that there is no column capturing the latest/highest time for all individuals. Column Arrive (Arrival <=> finish line) capture only finisher (status =1). Non-finisher individuals (status = 0) have only the last time corresponding to the time where they stop the race. Therefore, we create a new column a new column HighestTime to capture the information about the time-to-event regardless the status.
